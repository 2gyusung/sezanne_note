# 마크업 기본 문법 규칙
```
1. 요소는 시작/종료 태그가 있어야 한다.
   태그 중에 종료 태그가 없는 것 : 빈요소(hr, meta, br, img ,input , link)

2. 요소는 제대로 중첩이 되어야 한다
    <p>중첩된 요소가 있을 때는 <string>바르게</string>표현</p>

3. img 태그에는  alt 속성이 있어야 한다

4 권장하는 사항
    태그 속성은 소문자로 마크업 하는 것을 권장한다.

5 주석처리 방법
    협업 시 현재 작업한 내용을 쉽게 알아볼 수 있도록 도와주는 설명문
    주석내용은 브라우저 화면에 출력되지 않음

    시멘틱 : 의미론적인 마크업 할 때 의미에 부합하는 태그 사용
             
논리적 순서 마크업
       : 논리적 순서에 의해 마크업 하는 것

- 블록레벨 요소와 인라인 요소의 특징

    블록레벨 요소 : 블록레벨 요소는 줄바꿈이 일어난다.
                   블록레벨 텍스트와 인라인요소를 자식요소로 포함할 수 있다
                   블록레벨 요소 중에는 블록레벨 요소를 자식요소로 포함할 수 있는 요소와 포함할 수 없는 요소가 있다.
    
    인라인 요소 : 인라인 요소는 줄바꿈이 일어나지 않는다.
                 인라인 요소는 텍스트(문자)와 인라인 요소를 자식 포함 할 수 있다.
                 인라인 요소는 블록레벨 요소를 자식요소를 포함 할 수 없다.

 - <br>태그는 줄바꿈 태그이다.

  EntityCode : 특수문자 정리 코드표
```
